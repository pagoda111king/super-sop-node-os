# Memory Symbols

This project uses node symbols as compressed memory objects.

## Why

The goal is not decorative icons. The goal is fast recall and fast reasoning.

Each node has:

```text
rune + color + exaggerated image + motion + formula
```

## 14 Symbols

| Node | Rune | Symbol Image | Formula |
|---|---|---|---|
| Intake | `⟐` | swallowing star gate | messy signal -> task object |
| Context | `▣` | blue memory vault | task + history -> context pack |
| Radar | `◉` | one-eyed radar tower | question -> evidence |
| Normalize | `▽` | crystal funnel | messy evidence -> JSON |
| Data | `▦` | stone database matrix | query need -> storage design |
| Planning | `⌖` | golden compass | goal -> steps + dependencies |
| Routing | `⟡` | purple branch switch | state -> path |
| Learning | `✶` | pink question theater | concept -> teachable lesson |
| Action | `⚡` | green lightning hammer | plan -> real action |
| Test | `⚗` | bubbling lab flask | claim -> reproducible evidence |
| Score | `◎` | orange judgment gauge | evidence + tests -> verdict |
| Check | `⛨` | red shield gate | artifact -> pass/fix/return |
| Handoff | `⇄` | bronze bridge package | artifact -> person/system/agent |
| Sediment | `✺` | seed in strata | experience -> compound asset |

## Design Rules

- Use the rune before reading the node title.
- Bind each rune to a vivid image.
- Use emotion ethically: danger, humor, surprise, bodily motion, attraction.
- Avoid explicit sexual imagery in public/product contexts.
- Keep colors stable within one system.
